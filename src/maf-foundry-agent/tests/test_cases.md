# テストケース集

同ディレクトリの `sample_sales_data.csv`（動作検証用サンプル）をアップロードした状態で、
以下の質問を投げて検証する。狙いは **(a) Code Interpreter で実データを計算してから答えるか、
(b) 出力が事実として正しいか** の2点。アプリは任意の CSV/Excel に対応すること（列名の
ハードコード等は禁止。サンプルはあくまでテスト用）。

サンプルの列（1702行 / 13列）:
`order_id, order_date, region, channel, customer_id, customer_segment, category,
product_id, product_name, unit_price, quantity, discount_rate, sales_amount`

`quantity` / `customer_segment` / `sales_amount` に欠損を意図的に混入。日付範囲は
2023-01-01〜2024-12-31。

## 単発（Q1–Q10）

各ケースは独立したセッションで実行する。

| # | 質問 | 検証すべき事実 |
|---|---|---|
| Q1 | データの概要を教えて | 行数1702、列数13、欠損列(quantity/customer_segment/sales_amount)に言及（日付範囲 2023-01-01〜2024-12-31 は出れば望ましいが、汎用アナライザのため必須にはしない） |
| Q2 | 地域別の売上合計を教えて | 8地域すべてが返る（関東/関西が上位） |
| Q3 | 2024年の月次売上推移を可視化して | 2024でフィルタし折れ線を生成（plotコードが走る） |
| Q4 | 売上が最も多い商品TOP5は？ | 上位5件、合計額が併記（数値が5つ以上） |
| Q5 | 関東地域の最近3ヶ月の傾向を分析して | 関東に言及＋増減/変化率などの傾向シグナル |
| Q6 | カテゴリ別売上の比較を棒グラフで | 5カテゴリ(家電/食品/衣料品/書籍/雑貨)の棒グラフを生成 |
| Q7 | 数値項目の相関を分析して | sales_amount と unit_price/quantity の相関に言及 |
| Q8 | 顧客セグメントごとの平均購買額の違いは？ | VIP/リピーター/新規が併記され、**平均購買額で VIP が最上位** |
| Q9 | 2023年と2024年の同月比較 (YoY) | 2023/2024両方に言及＋成長率(%)、可視化 |
| Q10 | 異常値や欠損値の状況は？ | quantity/customer_segment/sales_amount の欠損に言及 |

> Q8補足: 実データでは `VIP ≈ 21,006 / 新規 ≈ 19,608 / リピーター ≈ 18,508` 円/件。
> リピーターと新規は接近しており反転しうるため、strict には「VIP が最大」のみを検証する。

## マルチターン（M1–M3）

1つの `AgentSession` 内で前ターンの状態（フィルタ済み df / 直前の集計・話題）を引き継ぐ。

- **M1**: 「2024年の関東地域に絞って」→「月別の売上推移を折れ線グラフにして」
  - T2 でフィルタを継続（再フィルタしても回答が 2024/関東 にスコープされていること）。
- **M2**: 「カテゴリ別売上の合計を多い順に教えて」→「その上位3カテゴリについて、地域ごとの売上を比較して」
  - T1 の上位3カテゴリが T2 に引き継がれること（コード内リテラル or 動的抽出）。
- **M3**: 「VIP顧客のみに絞って平均購買額を出して」→「フィルタを戻して、全体の平均購買額と比較して」
  - フィルタがリセットされ、最終回答に VIP値と全体値の両方が併記されること。
